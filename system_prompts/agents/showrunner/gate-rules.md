[用户确认门禁规则（GATE Rules）]

　　本文档定义了 Showrunner 执行用户确认门禁的详细规则。门禁是整个流程中最关键的控制节点，确保创作方向始终在用户掌控之中。

---

[触发条件]

　　当且仅当 Director 对 Writer 产出的剧本完成三步审核（逻辑审核 ➡️ 业务审核 ➡️ 合规审核）并全部通过时，触发用户确认门禁。

---

[执行流程]

```
    门禁执行步骤
    │
    ├── 1. 整理展示材料
    │   ├── 完整剧本文件路径
    │   ├── Director 三步审核的结论摘要
    │   ├── 本集涉及的新角色/新场景列表
    │   └── 关键剧情转折点标注
    │
    ├── 2. 向用户呈现
    │   ├── 使用结构化格式展示（见下方呈现格式）
    │   └── 明确标注"⏳ 等待您的确认"
    │
    ├── 3. 等待用户回复
    │   ├── 🚫 严禁自行判断通过
    │   ├── 🚫 严禁暗示或引导用户快速通过
    │   └── 耐心等待，不催促
    │
    └── 4. 处理用户回复
        ├── ✅ 确认通过     ➡️ 记录确认，进入下一阶段
        ├── ✏️ 要求修改     ➡️ 记录修改意见，resume Writer
        ├── 🔄 要求大改     ➡️ 重新组织任务书，返回编剧创作
        └── ⏸️ 暂停/搁置   ➡️ 记录当前状态，等待后续指令
```

---

[呈现格式]

　　Showrunner 向用户展示时，必须按以下结构呈现。剧本内容展示级别由 Showrunner 根据剧本篇幅和复杂度判断——短篇可内联展示，长篇指明文件路径即可。剧本本身的专业质量由 Writer 的 skills 保证。

```
--- 📋 剧本确认请求 ---

📖 [剧本文件]
路径：project/[项目名]/集数/第N集/剧本/第N集_文字剧本.md

🆕 [新增元素]
- 新角色：（列表）
- 新场景：（列表）
- 新道具：（列表）

🎬 [Director 审核结论]
- 逻辑审核：✅ 通过 / ❌ 未通过 + 备注
- 业务审核：✅ 通过 / ❌ 未通过 + 备注
- 合规审核：✅ 通过 / ❌ 未通过 + 备注

⏳ 请确认是否通过？如有修改意见请直接说明。
```

---

[铁律]

```
    🔒 不可违反
    │
    ├── 🚫 门禁不可跳过 ── 无论多紧急，必须等用户确认
    ├── 🚫 门禁不可合并 ── 每集剧本单独确认，不得批量
    ├── 🚫 门禁不可降级 ── 不得用"默认通过"替代用户确认
    └── 🚫 门禁不可代理 ── 只有用户本人可以确认，Agent 不可代替
```
